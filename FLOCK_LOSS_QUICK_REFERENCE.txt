ISSUE FIXED
───────────────────────────────────────────────────────────────────────────────
  ❌ BEFORE: Recording a flock loss didn't update bird records
           → Headcount remained incorrect
           → Users had to manually mark birds as deceased
           → Data inconsistency

  ✅ AFTER:  Recording a flock loss automatically updates bird records
           → Headcount updated immediately
           → No manual work needed
           → Data stays consistent


HOW IT WORKS
───────────────────────────────────────────────────────────────────────────────

  User Records Loss          App Auto-Updates           Result
  ─────────────────          ────────────────           ───────
  5 birds lost               5 birds marked             Headcount:
  to predator                as deceased                50 → 45 ✅


KEY FEATURES
───────────────────────────────────────────────────────────────────────────────

  📋 Info Banner
     "The specified quantity of active birds will automatically be marked as
      deceased"
     • Only shown when creating NEW loss (not editing)
     • Updates based on loss type (deceased vs sold)
     • Blue color (non-alarming)

  💾 Automatic Bird Status Updates
     • Records loss → Marks birds as "deceased"
     • Sold loss type → Marks birds as "sold"
     • Other types → Marks birds as "deceased"
     • Uses database transaction for consistency

  ⚠️  Delete Warning
     "Note: bird records marked as deceased will not be unmarked"
     • Explains that deleting loss doesn't change bird status
     • Helps users understand data integrity

  🔄 Data Consistency
     • Edit mode: Does NOT re-mark birds (intentional)
     • Delete mode: Does NOT revert bird status (intentional)
     • New losses: Auto-mark birds (automatic)

  other           →    deceased      Unknown cause
  sold            →    sold          Sold to customer


DATA CONSISTENCY RULES
───────────────────────────────────────────────────────────────────────────────

  ✅ AUTOMATIC (Bird Status Updates)
     • Recording a NEW loss
     • Creating multiple birds from purchase
     • Manual bird status changes

  ❌ INTENTIONAL (No Update)
     • Editing a loss (change quantity, type, date)
     • Deleting a loss
     • Recording same loss twice

  WHY?
  • Edit doesn't re-mark: Prevents accidental changes. Delete + re-record if needed.
  • Delete doesn't revert: Bird is still actually gone. Separate history.
  • Only new losses: Old data stays as-is. Only new losses use auto-update.
